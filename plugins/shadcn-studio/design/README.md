# shadcn-studio — staging initiative design docs

Design records for the multi-piece initiative that adds an always-on, deep,
interactive staging area to the taskmaster brainstorm pipeline. Each piece was
shaped through a `/taskmaster:brainstorm` dialogue and red-teamed across three
lenses before build. Preserved here (tracked) because `taskmaster-docs/` is
gitignored.

| # | Doc | Status | Ships in |
|---|-----|--------|----------|
| 1 | [01-engine-piece1.md](./01-engine-piece1.md) — greenfield interactive shadcn engine | Built | `shadcn-studio` (skill `studio` + template) |
| 2 | [02-brainstorm-staging-piece2.md](./02-brainstorm-staging-piece2.md) — always-on staging doctrine | Built | `taskmaster` (skill `brainstorm`) |
| 3 | [03-lanes-depth-dataviz-piece3.md](./03-lanes-depth-dataviz-piece3.md) — lanes, depth matrix, dataviz | Built | `shadcn-studio` (skill `deep-staging` + template) |
| 4 | [04-durable-visual-contract-piece4.md](./04-durable-visual-contract-piece4.md) — durable visual contract into the spec/cards | Built | `taskmaster` (skill `visual-contract` + grill/task-cards/spec-adversary/spec-redteam/coverage-check) |
| 5 | brainstorm question + design-doc quality | Not designed | — |

The three built pieces form a usable slice: a greenfield interactive shadcn
sandbox (1), brainstorm that always stages design and creative options and routes
fidelity (2), and lane/depth/dataviz authoring doctrine so staged options carry
substance (3). Piece 4 binds those decisions into the spec and cards as a durable
contract. Piece 5 (brainstorm question + design-doc quality) is named, not designed.
